# 🔐 인증 시스템 검토 문서

## 개요
JWT 토큰 기반 인증 시스템 검토

## 인증 방식

### JWT 토큰 구조
```json
{
  "id": 1,
  "authType": "social",
  "iat": 1753423558,
  "exp": 1753427158
}
```

### 토큰 사용 방법
- **REST API**: `Authorization: Bearer YOUR_JWT_TOKEN`
- **WebSocket**: `auth: { token: 'YOUR_JWT_TOKEN' }`

## 검토 포인트

### ✅ 확인할 사항
- [ ] JWT 토큰 만료 시간 적절성 (현재: ~1시간)
- [ ] 토큰 갱신 메커니즘 구현 여부
- [ ] 토큰 검증 로직 보안성
- [ ] authType 필드 활용 방안

### ⚠️ 보안 고려사항
- [ ] 토큰 저장 방식 (LocalStorage vs HttpOnly Cookie)
- [ ] HTTPS 필수 여부 확인
- [ ] 토큰 탈취 방지 대책
- [ ] 리프레시 토큰 필요성 검토

### 🔍 테스트 시나리오
1. **유효 토큰 테스트**
   - 정상적인 API 요청 처리 확인
   
2. **만료 토큰 테스트**
   - 401 에러 반환 확인
   - 자동 갱신 또는 로그아웃 처리
   
3. **잘못된 토큰 테스트**
   - 403 에러 반환 확인
   - 보안 로그 기록 확인
